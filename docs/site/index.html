The problem

AI agents hold keys.
There is no recovery path.

Autonomous agents manage wallets, sign transactions, and control real assets. When a key is compromised, every existing solution falls short.

🏦

Centralized key vaults

Single point of failure. One breach compromises every agent connected to it.

🔄

Manual rotation

Doesn't scale. Requires human intervention at exactly the worst moment.

📄

Backup keys in config

Stored in plaintext. The first thing an attacker looks for after initial compromise.

💀

No recovery at all

Lose the key, lose the agent. All on-chain state and reputation gone.

How it works

Trustless recovery,
in three steps

Spectre does not deploy agents — it protects existing ones. Register a recovery config, then recover when you need to.

01

Register

Owner registers a recovery config for any existing agent key. Stored on-chain: email hash, identity commitment, guardian list, timelock duration. The agent is unchanged.

02

Initiate recovery

M-of-N guardians sign a recovery transaction with the new owner key. Timelock starts. The current owner can cancel at any point during the window using their existing key.

03

Execute

After the timelock expires, anyone can finalize. The agent cell gets the new owner key. Nonce increments. The old key can no longer spend the cell.

recovery flow
Agent cell (agent-lock + agent-type) │ │ Guardian sigs → sets pending_owner_pubkey ▼ Recovery-pending cell (recovery-lock + agent-type) │ ├── Owner cancels → restored agent cell, nonce + 1 │ └── Timelock expires (2880 blocks ≈ 6.4h on testnet) → new agent cell with new owner key
